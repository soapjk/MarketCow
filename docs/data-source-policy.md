# 数据源与再分发政策

## 代码许可证不等于数据授权

本项目的代码许可证只覆盖本仓库中有权许可的代码和文档。它不授予对交易所数据、网站内容、行情、财报、指数、商标或第三方客户端所访问数据的任何权利。

“免费访问”也不等于可商用、可批量抓取或可再分发。使用者必须分别核对每个来源当前的服务条款、robots 规则、频率限制、地域限制和数据许可证。

## Provider 接入原则

- 只访问无需绕过认证、付费墙、验证码或技术访问控制的端点。
- 使用可识别的 User-Agent，控制并发和频率，并对 429、封禁和字段漂移明确失败。
- 保存来源 URL、观测时间、入库时间和原始响应定位信息。
- 原始响应仅用于本地复现和审计，不由本项目提供下载或再分发。
- 不在仓库、测试样本或 issue 中提交访问令牌、账户数据、个人信息或大规模上游数据。

## 来源登记

新增 provider 的贡献必须记录访问方式、数据范围、已知限制和条款风险。来源条款发生变化时，应优先停用有疑问的采集路径，再完成复核，而不是默认继续运行。

Mootdx 等第三方客户端的代码许可证与其文档中声明的使用限制可能不是同一问题。默认安装和使用这些客户端前，应阅读对应项目和实际数据来源的最新条款。Yahoo、新浪、东方财富、通达信及其他来源同样需要独立判断。

## 发布边界

公开源码时不得附带 `data/`、DuckDB、Parquet、原始响应、日志、缓存或凭证。本项目默认只为本地个人研究提供工具，不承诺任何来源适用于商业服务或面向公众的数据 API。
